## Learning Path

1. **Foundation**: Learn core principles and terminology.
2. **Implementation**: Build small working examples in Godot/GDScript.
3. **Deep dive**: Explore engine internals and advanced patterns.
4. **Production**: Polish gameplay, performance, and deployment.

## Common Pitfalls

- Misusing node parenting and scene composition.
- Forgetting to use `delta` for frame-independent movement.
- Writing large monolithic scripts instead of small behaviors.
- Ignoring the editor’s built-in tools like AnimationPlayer and TileSet.
- Hardcoding input and not using the InputMap.

## Best Practices

- Keep scripts small, single-responsibility, and well-named.
- Prefer signals over tight coupling.
- Use groups and autoloads sparingly and intentionally.
- Optimize only after profiling with Godot’s debugger/profiler.
- Follow Godot naming conventions and scene structure.
