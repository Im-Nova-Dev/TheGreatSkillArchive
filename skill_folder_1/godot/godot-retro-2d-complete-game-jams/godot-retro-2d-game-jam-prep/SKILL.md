# Godot Retro 2D Game Jam Prep

## Core Concepts
Prep checklist for a one-week retro 2D jam game in Godot.
- Day 1: pick one genre and one art style.
- Day 2: core movement and one enemy.
- Day 3: one level + one mechanic + one boss or goal.
- Day 4: juice (screen shake, hitstop, particles, sound).
- Day 5: UI, game over, credits, export.
- Day 6: bug bash + trailer GIF.
- Day 7: submit.

## GDScript Check
```gdscript
# Always keep a working playable build:
# - Run the project every hour.
# - Keep a changelog of tested states.
# - Lock core physics before adding art.
```

## Common Pitfalls

- Overcomplicating scope before the core loop is confirmed fun.
- Polishing visuals before gameplay feels right.
- Missing one-click export workflow until submit day.
- Ignoring Godot’s built-in TileMap and Animation tools.

## Best Practices

- Scope to one screen first.
- Profile every build.
- Ship one polished level instead of three rough ones.
- Keep art in consistent palette and resolution.
