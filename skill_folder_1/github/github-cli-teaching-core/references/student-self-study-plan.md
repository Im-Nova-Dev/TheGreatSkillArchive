# Self-Study Plan — Git and GitHub

Weekly plan assuming 30–60 minutes a day.

## Week 1: Local Git Only

Goal: never feel afraid of `git status`.

Day 1 — create repo, commit files, review with log
Day 2 — branches: create, switch, rename, delete
Day 3 — staging strategies: `git add`, `git add -p`, reset progression
Day 4 — diff and history: `git diff`, `git show`, `git log`
Day 5 — accidents: restore files, undo commit, reflog recovery

## Week 2: Push and Pull

Goal: keep local and GitHub in sync

Day 1 — clone a repo and inspect it
Day 2 — add remote, push new commits
Day 3 — pull, fetch, merge updates
Day 4 — conflicting changes
Day 5 — push to GitHub and view changes

## Week 3: Branches and PRs

Goal: complete the review cycle

Day 1 — create a feature branch and push
Day 2 — open PR, inspect diff and checks
Day 3 — leave a review, request changes
Day 4 — fix feedback and push
Day 5 — merge PR, delete branch, clean state

## Week 4: Conflict and Recovery

Goal: handle messy history calmly

Day 1 — read conflict markers and resolve
Day 2 — merge conflict on purpose and repair
Day 3 — reflog recovery drill
Day 4 — amend and interactive rebase
Day 5 — cherry-pick between branches

## Week 5: GitHub in Depth

Goal: use GitHub features efficiently

Day 1 — issues and labels
Day 2 — milestones and projects overview
Day 3 — Actions workflows
Day 4 — releases and assets
Day 5 — secrets and branch protection

## Week 6: Collaborative and Open Source

Goal: work across repos

Day 1 — fork + upstream setup
Day 2 — sync upstream changes
Day 3 — contribute to a real small repo
Day 4 — respond to community review
Day 5 — reflect on workflow and write a personal cheat sheet

## Suggested cadence

- Study 30–60 minutes daily
- End each week with a real PR (even tiny)
- Keep a learning journal of commands you used
