# Warm-Up Quiz — Git and GitHub in the Terminal

Use these questions as check-for-understanding exercises before moving to the next lesson.

## Beginner

1. What is Git, in one sentence?
2. What is the difference between GitHub and Git?
3. What does `git status` show?
4. What is the staging area (index)?
5. What does `git commit` do?
6. Does `git commit` upload anything to GitHub?
7. What does `git push` do?
8. How do you create a new branch?
9. How do you switch branches?
10. How do you see recent commits?

## Intermediate

11. What is the difference between `git pull` and `git fetch`?
12. What does `git diff main...HEAD` show?
13. When would you use `git rebase`?
14. What is a merge commit?
15. What is a PR?
16. How do you inspect failed CI logs with `gh`?
17. What is the `.git` folder?
18. What does `git reflog` do?
19. What is a “triangular” workflow?
20. What is an upstream remote?

## Advanced

21. How do you create a GitHub release from the terminal?
22. How do you set a GitHub Actions secret?
23. What GraphQL or REST endpoints create and merge a PR?
24. How do you enable auto-merge via `gh`?
25. What does `git cherry-pick` do?
26. How do you change history without touching shared branches?
27. How do you find the GitHub API rate limit?
28. What is the difference between `git checkout` and `git restore`?
29. How do you create a GitHub release asset with `curl`?
30. What is branch protection and when would you use it?

## Suggested delivery

- Ask verbally and wait for an answer.
- If they answer partially, refine instead of correcting immediately.
- For wrong answers, ask a follow-up question that leads them to the right idea.
- Score is less important than whether they can explain it in their own words.
