# Pair Programming Prompts — Git and GitHub

Use these during live pairing sessions. Alternate roles often.

## Session A — First Commit Flow

- Driver: initialize repo
- Navigator: explain each command before it runs
- Switch every 2 minutes

Goal: learner understands working tree → staging → commit.

## Session B — Branch Workflow

- Driver: create branch and make changes
- Navigator: inspect state with `git status` and `git log`
- Switch every 2 minutes

Goal: learner understands branches move independently.

## Session C — PR Review

- Driver: open PR
- Navigator: read diff, leave review
- Switch every 2 minutes

Goal: learner understands PR = conversation.

## Session D — Conflict Resolution

- Driver: create conflict
- Navigator: guide resolution
- Switch every 2 minutes

Goal: learner reads markers, decides content, resolves.

## Prompt Cards

Shuffle these and pick one when you need a mini-exercise.

- “Amend the last commit without changing its message.”
- “Rename the current branch.”
- “Undo staged changes only.”
- “Create a PR and request a review.”
- “View the last three commits using one command.”
- “Delete a local branch safely.”
- “Undo a commit while keeping the changes.”
- “Show changed files without diff details.”
- “Switch branches while discarding uncommitted work.”
- “Find the last commit that touched README.md.”
- “Open a GitHub issue from the CLI.”
- “View CI logs for a failed run.”
