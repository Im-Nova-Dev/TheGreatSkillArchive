# Fork-Based Contribution Workshop

## Flow
1. Fork upstream repo on GitHub
2. Clone your fork locally
3. Add upstream as remote
4. Branch, commit, push to your fork
5. Open PR from fork to upstream
6. Respond to review
7. Sync fork with upstream regularly

## Key Commands
```
git remote add upstream <original-repo-url>
git fetch upstream
git rebase upstream/main
```

## Pitfalls
- Forgetting to sync before opening PR
- Editing fork directly on GitHub and also locally
