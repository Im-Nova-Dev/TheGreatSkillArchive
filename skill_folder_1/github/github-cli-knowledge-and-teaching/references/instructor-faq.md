# Instructor FAQ — Teaching Git and GitHub

## What if someone falls behind?

Do not slow the whole group. Assign them a helper or pair them with a faster learner. Return to them later during practice time.

## What if the projector fails?

Have a printed command list ready. Ask learners to follow along on their own machines while you narrate.

## What if GitHub is down?

Teach Git-only commands: init, add, commit, status, log, branch, diff. Save GitHub topics for later.

## What if someone already knows Git?

Give them a challenge task: clean history, review a PR, write a workflow YAML, or help a peer.

## What if the learner is scared of breaking things?

Do safety exercises:
- “break and recover” drills
- show `git reflog`
- show `git restore`
- explain that Git rarely loses data permanently

## What if someone asks why not use a GUI?

Valid. The goal here is terminal literacy, not GUI rejection. Explain that `gh` makes GitHub fast in the terminal; choice is fine later.

## What if authentication fails?

Have a fallback plan:
- HTTPS token method
- SSH method
- `git` + `curl` method

## What if the group is mixed skill level?

Use stations:
- beginner station: first commit and branch
- intermediate station: PR review and merge
- advanced station: fork workflow and Actions

Rotate every 20 minutes.
