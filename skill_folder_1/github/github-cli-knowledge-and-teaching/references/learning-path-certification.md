# Git and GitHub Learning Path — Certification Track

Use this to map progress from beginner to competent.

## Level 1 — Foundations

- Install Git and set identity
- Create a repo and make commits
- Push to GitHub and create a PR
- Read and write basic commit messages
- Recognize the working tree, staging area, and `.git` folder

## Level 2 — Branching

- Create, switch, rename, and delete branches
- Merge a feature branch into `main`
- Resolve a basic merge conflict
- Describe the difference between merge commits and fast-forward
- Use `git log --graph --oneline --all` to inspect history

## Level 3 — Collaboration

- Review a PR with `gh pr review`
- Respond to review feedback and update the PR
- Use `git fetch` and `git rebase` to keep a branch current
- Use `git cherry-pick` intentionally
- Recover from mistakes using `git reflog`

## Level 4 — GitHub Features

- Create and manage issues
- Create releases and add assets
- Configure a simple Actions workflow
- Manage secrets and artifacts
- Use branch protection rules

## Level 5 — Advanced Git

- Perform interactive rebases safely
- Rewrite history with `git filter-repo` on private branches only
- Use `git bisect` to identify regressions
- Explain objects: blob, tree, commit, tag
- Use `git worktree` for multi-branch workflows

## Certification Checkpoints

- Checkpoint 1: first merged PR from local-only work
- Checkpoint 2: 5 non-trivial merges without help
- Checkpoint 3: successful upstream open-source contribution
- Checkpoint 4: clean history rewrite on a private branch
- Checkpoint 5: authored a workflow and a release published to GitHub

## Suggested Badges

- “First Commit” after Checkpoint 1
- “Branch Boss” after Checkpoint 2
- “Open Source Contributor” after Checkpoint 3
- “History Editor” after Checkpoint 4
- “Pipeline Author” after Checkpoint 5
