# Git Security Workshop

## Why security matters
Secrets in Git history are exposed forever. Learn to prevent, detect, and remediate.

## Topics
1. `.gitignore` best practices
2. Scanning for secrets (truffleHog, git-secrets)
3. Removing secrets from history (BFG, filter-repo)
4. GitHub secret scanning and Dependabot
5. SSH key hygiene

## Exercises
1. Add common secret patterns to `.gitignore`
2. Scan a sample repo with `trufflehog`
3. Practice removing a file from history

## Discussion
What’s worse: a leaked password or a messy history?
