# Progressive Demo Progression

Use this as a plan for a sequence of live demos that increase in complexity.

## Progression 1 — The Simplest Repo

- `git init`
- one file, one commit
- `gh repo create` and `gh browse`

## Progression 2 — Second Commit and Push

- second commit
- `git push`
- PR create and view

## Progression 3 — Branch and Switch

- create branch
- edit, commit
- switch back
- show file reverts

## Progression 4 — Merge

- merge branch
- show merge commit
- show fast-forward

## Progression 5 — Conflict

- conflicting edits
- conflict markers
- resolution

## Progression 6 — Rebase

- rebase onto main
- show linear history
- explain rewrite risk

## Progression 7 — Fork

- fork upstream
- add upstream remote
- sync upstream
- upstream PR

## Progression 8 — CI and Review

- failing workflow
- log review
- fix and green

## Progression 9 — Release

- `gh release create`
- asset upload
- tag visualization

## Progression 10 — Full Cycle

- idea -> branch -> commits -> PR -> review -> merge -> release -> cleanup
