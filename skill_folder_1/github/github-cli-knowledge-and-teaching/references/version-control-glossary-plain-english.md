# Version Control Glossary — Plain English

**Repository (repo):** A folder that remembers every change ever made inside it.

**Commit:** A saved snapshot of the project at a point in time.

**Branch:** A parallel timeline you can experiment on without breaking the main one.

**Merge:** Joining two timelines together.

**Conflict:** Git found the same lines changed in both timelines and needs you to pick the truth.

**Remote:** A copy of your repo hosted online (like GitHub).

**Clone:** Downloading a remote repo to your machine.

**Push:** Uploading your changes to a remote.

**Pull:** Downloading changes from a remote and merging them.

**Pull Request (PR):** A request to merge your branch into someone else’s branch, with discussion.

**HEAD:** The “you are here” marker in your repo.

**Tag:** A name attached to a specific commit — useful for versions.

**Reflog:** Git’s personal diary. Use it when you think work is lost.
