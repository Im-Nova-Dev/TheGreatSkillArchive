# Branch Strategy Tournament

Compare three strategies head-to-head.

## Strategies
1. **GitHub Flow**
   - Main always deployable
   - Feature branches → PR → merge

2. **Git Flow**
   - main, develop, feature, release, hotfix

3. **Trunk-Based Development**
   - Short-lived branches
   - Feature flags for incomplete work

## Tournament Format
- Round 1: explain each strategy in 2 minutes
- Round 2: argue for one in a debate
- Round 3: vote on best fit for given scenario

## Scenarios
- Solo developer releasing monthly
- 20-person team shipping daily
- Open-source project with volunteers
