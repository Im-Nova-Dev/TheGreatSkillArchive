# Git for Designers Module

A shorter, gentler track for designers, PMs, and non-developers.

## Session A — Git in Plain English
- Git = design history log for text
- Commit = save a version
- Branch = make a copy to explore without breaking the original

## Hands-On
- Add a README with a dark-mode color palette.
- Commit as “chore: add dark palette”.
- Create branch `experiment/neon` and modify a CSS value.
- Merge it with a merge commit and show the comparison.

## GitHub for Designers
- Issue = design ticket
- PR = prototype proposal
- Review = comment in context
- Release = publish version for users

## Outcome
- Comfortable reading PR diffs
- Can comment on a design implementation PR
- Understands “can you review this?” workflow
