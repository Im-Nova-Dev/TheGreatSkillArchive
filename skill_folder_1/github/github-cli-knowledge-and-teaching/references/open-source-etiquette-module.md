# Open-Source Etiquette Module

## Unwritten Rules
- Read CONTRIBUTING.md before opening a PR
- Start small: docs and typo fixes
- Be patient — maintainers are often volunteers
- Use “good first issue” labels as entry points
- Thank reviewers for their time

## Exercises
- Read a project’s CONTRIBUTING.md and summarize the workflow
- Find a “good first issue” and draft an approach
- Practice writing a non-demanding PR description
- Role-play a review exchange
