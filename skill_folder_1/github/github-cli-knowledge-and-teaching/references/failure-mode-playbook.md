# Failure Mode Playbook

When things go off-script — intentional adaptations.

## Scenario 1: Majority lagging
- **Fix**: Introduce “buddy system” + reduce exercise scope to one command
- **Later**: Assign homework to finish earlier exercise batch

## Scenario 2: Room too quiet
- **Fix**: Poll with emoji reactions, ask rhetorical question
- **Later**: Break into pairs, then report back

## Scenario 3: Setup failure cluster
- **Fix**: Project your terminal, run class as shared screen
- **Later**: Provide VM / codespace as fallback for next session

## Scenario 4: One advanced learner dominates
- **Fix**: Remind them to act as assistant, not instructor
- **Later**: Give them extra exercises to keep busy

## Scenario 5: Too many questions at once
- **Fix**: Capture questions on shared doc, address after the break
- **Later**: Add “parking lot” column to slide

## Scenario 6: Learner upset over mistake
- **Fix**: Normalize: “I broke something worse last week.”
- **Later**: Do a “fail forward” demo next session
