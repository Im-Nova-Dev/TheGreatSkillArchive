# Practice Rubric for Git Assignments

Use this to give structured feedback on student Git work.

## Areas

1. Commit quality
2. Branch usage
3. PR description quality
4. Review participation
5. Recovery behavior

## Rubric

### Commit Quality

- Excellent: atomic commits, clear messages, logical sequence
- Good: mostly clear messages, minor cleanup needed
- Needs work: large commits, vague messages, unrelated changes mixed

### Branch Usage

- Excellent: branch per task, clean merges, no force push to shared branches
- Good: correct branching, occasional messy history
- Needs work: commits on main, abandoned branches, force push without care

### PR Description Quality

- Excellent: context, motivation, test plan, clear summary
- Good: title and basic description present
- Needs work: missing description, no context, no reviewer guidance

### Review Participation

- Excellent: constructive feedback, references code lines, clear verdict
- Good: basic approval or request-changes
- Needs work: no review or non-constructive comments

### Recovery Behavior

- Excellent: uses reflog, amend, or rebase intentionally without breaking shared history
- Good: recovers with help
- Needs work: afraid to fix, leaves broken history untended

## Usage

Review each area after assignment. Provide one actionable improvement for the next task.
