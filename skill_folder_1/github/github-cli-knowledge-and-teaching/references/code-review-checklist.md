# Code Review Checklist

## Before Reviewing
- [ ] Read PR title and description
- [ ] Check linked issue
- [ ] Run tests locally if needed (`git checkout <pr-branch>`)

## Content
- [ ] Does the code solve the problem?
- [ ] Are edge cases considered?
- [ ] Are tests added / updated?
- [ ] Is documentation updated?
- [ ] Are there no unnecessary changes?

## Communication
- [ ] Tone is kind and specific
- [ ] Line comments are precise
- [ ] Approval / change request is clear

## After Reviewing
- [ ] Re-check after author updates
- [ ] Approve when ready
