# Progressive Git Exercises

Exercises ranked by difficulty. Start where confidence is high.

## Level 1 — Complete These Without Help

1. Create a new repo, add one file, commit it, and run `git log`
2. Create a branch, add a file, commit, switch back to `main`
3. Stage one file, unstage one file, check status
4. Edit a file, commit, then change it and see diff
5. Push an empty repo to GitHub and view it in the browser

## Level 2 — Guided Practice

1. Create two branches, edit same file, merge one branch, then merge the second branch and resolve a conflict
2. Amend last commit message without changing the file
3. Use `git reflog` after deliberately moving HEAD
4. Recover a commit by cherry-pick after resetting a branch
5. Rebase a private branch onto latest `main`

## Level 3 — Open-Ended Practice

1. Create a team workflow document for a three-person project using Git
2. Set up branch protection rules and auto-merge on a test repo
3. Write a 5-step GitHub Actions workflow for lint and test
4. Mentorship: walk a peer through their first PR review
5. Contribute a small fix to an open-source repository you use

## Level 4 — Challenge Scenarios

1. History archaeology: use `git bisect` to find the commit that introduced a regression
2. Large rebase: clean up 20+ commits into a readable sequence without losing work
3. Multi-repo sync: maintain a fork across upstream updates for one month
4. Emergency recovery: restore a repo after accidental `git reset --hard`
5. Automation: write a shell script that runs lint, tests, and opens a PR via `gh` API

## Scoring

- Level 1: zero failures required
- Level 2: complete 4 of 5
- Level 3: complete 3 of 5
- Level 4: at least one successful challenge
