# Git for Writers and Docs Teams Module

## Why
Docs and prose change over time — Git tracks those changes.

## Workflow
1. Markdown files in a repo
2. Branch `docs/update-api`
3. Commit with message `docs: clarify authentication section`
4. Open PR, get review from a subject matter expert
5. Merge and deploy via Actions

## Exercises
- Fix a typo and open a PR
- Suggest an edit via inline comment
- Redirect a broken link and show the diff
