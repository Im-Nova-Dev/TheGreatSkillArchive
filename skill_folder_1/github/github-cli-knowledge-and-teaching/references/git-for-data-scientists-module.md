# Git for Data Scientists Module

## Why Git for Data
Version your notebooks, experiments, and models. Share analyses without emailing files.

## Core Concepts
- Notebooks are code + prose: treat them like source code
- Track model outputs as artifacts, not commits
- Use branches for experiment variants

## Walkthrough
1. Create repo for analysis project
2. Commit notebooks and requirements.txt
3. Branch `experiment/tuning` and iterate
4. Merge results back to main
5. Tag releases with model version

## GitHub Actions for Data
- Run notebook validation on PR
- Generate reports as artifacts
- Deploy dashboard to GitHub Pages
