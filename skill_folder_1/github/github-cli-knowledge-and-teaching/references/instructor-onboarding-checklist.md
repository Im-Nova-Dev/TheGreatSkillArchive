# Instructor Onboarding Checklist

Use this before teaching a cohort.

## Environment

- [ ] Git installed on instructor machine
- [ ] `gh` installed and authenticated
- [ ] GitHub account with permissions for demo repos
- [ ] Screen sharing and microphone tested
- [ ] Shared demo repo prepared
- [ ] Terminal font size readable for remote attendees

## Materials

- [ ] printed or digital one-page cheatsheet
- [ ] warm-up quiz ready
- [ ] teaching drills selected for first session
- [ ] troubleshooting sheet visible
- [ ] onboarding checklist for learners

## Lesson Plan

- [ ] define the session goal
- [ ] list the commands to demonstrate
- [ ] write the checkpoint question
- [ ] prepare error scenarios to show

## Before Each Session

- [ ] test all code blocks in a fresh terminal
- [ ] clear local state that could confuse learners
- [ ] confirm shared repo or shared view is accessible

## During Session

- [ ] teach the concept before the command
- [ ] show output after every command
- [ ] ask “what do you expect next?”
- [ ] normalize errors: show them and fix them live

## After Session

- [ ] collect feedback using template
- [ ] note commands that tripped people up
- [ ] update planned next session pacing
