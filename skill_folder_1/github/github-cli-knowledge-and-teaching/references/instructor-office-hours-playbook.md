# Instructor Office Hours Playbook

## Format
- 1 hour weekly
- Drop-in style; no sign-up needed
- Optional: one short “deep dive” topic per session

## Session Structure
1. **Welcome** — 2 min
2. **Triage** — collect questions; write on whiteboard
3. **Top-voted question first** — 15 min live debugging
4. **Deep dive** — 20 min specific topic e.g. “merge conflict resolution deconstructed”
5. **Open floor** — 20 min for ad-hoc help
6. **Wrap-up** — 3 min, summarize 3 takeaways

## Common Questions
- “Why did my push get rejected?”
- “Can I rebase after pushing?”
- “How do I undo a merge?”
- “Is git better than svn / perforce / my old tool?”

## Rules
- One learner speaks at a time
- Instructor narrates while typing
- Always ask “what do you expect to happen?” before fixing
- Document the question and answer for next cohort
