# Mentor Matching Guide

## Purpose
Pair each learner with an experienced Git/GitHub user for one-to-one guidance.

## Mentor Profile
- Has used Git daily for at least 1 year
- Can explain `git rebase` without Googling
- Available for 30-minute weekly slot

## Matching Criteria
- Same language if possible (but not required)
- Schedule overlap (timezone / availability)
- Interest overlap (web, data, systems)

## Mentor Script

1. Introduce the “buddy branch” pattern: both work in the same repo on separate branches.
2. Review the learner’s first PR together.
3. Demonstrate `git add -p` selective staging.
4. Walk through one real merge conflict resolution.
5. End with one thing to practice before next session.
