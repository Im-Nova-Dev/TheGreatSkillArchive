# Pull Request Storytelling

## Concept
A PR should read like a story: Why? What? How?

## Template
```
Why:
- [Problem / motivation]

What:
- [Summary of changes]

How:
- [Implementation notes]

Testing:
- [How to verify]

Screenshots / captures:
- [If applicable]
```

## Exercise
1. Pick a recent commit.
2. Rewrite its message as a PR description using this template.
3. Exchange with partner; improve based on feedback.

## Skill Check
- Does it explain motivation?
- Is it scannable?
- Does it link to an issue?
