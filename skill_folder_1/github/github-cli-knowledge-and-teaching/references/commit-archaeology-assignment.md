# Commit Archaeology Assignment

## Task
Given a sample repo, reconstruct the team’s decisions.

## Questions
1. What feature was added first?
2. When was the README updated?
3. Who rewrote history? (reflog / rebase clues)
4. What commit introduced a bug? Use `git log -S` or `git blame`.
5. How many merge commits vs fast-forwards?

## Deliverable
Short report: 3 findings + 1 question you still have.
