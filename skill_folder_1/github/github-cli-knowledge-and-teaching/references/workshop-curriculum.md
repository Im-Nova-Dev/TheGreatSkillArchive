# Git and GitHub Workshop Curriculum

A full-day workshop outline for hands-on learning.

## Schedule

- 09:00 Introduction and mental model
- 09:30 Git basics: init, add, commit, status
- 10:30 Break
- 10:45 Branches: create, switch, merge
- 12:00 Lunch
- 13:00 Remotes: push, pull, fetch
- 14:00 GitHub PRs: create, review, merge
- 14:30 Break
- 14:45 CI: read logs, fix failures
- 15:30 Forking and upstream workflow
- 16:00 Wrap-up, resources, next steps

## Learning Objectives

By the end, participants will be able to:
- Create and manage a Git repository locally
- Stage and commit changes meaningfully
- Branch, merge, and resolve basic conflicts
- Push to GitHub and create a pull request
- Review a pull request and leave feedback
- Diagnose common CI failure patterns
- Keep a fork in sync with an upstream repository

## Group Size

- Ideal: 10–20 participants
- Minimum: 5
- Maximum: 30 (requires additional helpers)

## Setup Requirements

- Each participant needs Git, `gh`, and a GitHub account
- Instructor needs a projector or screen sharing
- Shared repo for live demo prepared ahead of time
- Internet connectivity for GitHub access

## Assessment

Use warm-up quiz at start and end. Use drills to observe real skill during the day.
