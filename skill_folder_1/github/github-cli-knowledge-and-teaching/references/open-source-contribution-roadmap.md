# Open-Source Contribution Roadmap

## Beginner Repos
- `octocat/Spoon-Knife` (sandbox)
- Good-first-issue repos labeled for new contributors
- Documentation-only repos

## Intermediate
- Project with `CODE_OF_CONDUCT.md` and `CONTRIBUTING.md`
- Repos using `gh` and issue forms
- Smaller CLI tools or libraries

## Advanced
- Larger codebases with CI and tests
- Projects that require review cycles
- Any repo you use daily

## Workflow
1. Read docs and contributing guide
2. Pick an issue
3. Fork + branch + commit + push
4. Open PR with linked issue
5. Iterate on reviews
6. Celebrate merge

## Upkeep
- Add a “Contributor” section to your README
- Document your process
- Mentor someone else
