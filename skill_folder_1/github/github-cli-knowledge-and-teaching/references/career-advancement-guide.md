# Career Advancement Guide

## How Git/GitHub Skills Help
- Open-source contributions = portfolio
- Professional workflow knowledge = productivity multiplier
- CLI fluency = automation and scripting ability

## Mapping Skills to Roles
| Role | Must-Know Git Skills |
|------|----------------------|
| Frontend Dev | branches, PRs, rebase, merge |
| Backend Dev | same + Actions, releases |
| DevOps | tags, releases, Actions, workflows |
| Engineering Manager | PR review, branch strategy, protected branches |
| Technical Writer | PRs, markdown diffs, releases |

## Portfolio Ideas
- A well-maintained personal site deployed via Actions
- A public contribution to a well-known project
- A custom Git workflow documented in a README

## Interview Lines
- “I prefer `git pull --rebase` to keep history clean.”
- “I use `git add -p` to stage thoughtfully.”
- “I review PRs with a checklist, not just vibes.”
