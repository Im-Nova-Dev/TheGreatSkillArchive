# Git for Game Devs Module

## Why Games Are Different
- Large binary assets (textures, models)
- Multiple collaborators on same scenes
- Need for tagged releases and patches

## Git Strategies
- Use Git LFS for large binaries
- Splitting logic (text format) from assets (LFS)
- Tag releases for shipped versions

## GitHub Features
- Releases for patches and DLC
- Issues for bugs and feature requests
- Projects for milestone planning

## Demo
- Track a small game project
- Tag `v0.1.0` as first playable
- Release notes with patch notes
