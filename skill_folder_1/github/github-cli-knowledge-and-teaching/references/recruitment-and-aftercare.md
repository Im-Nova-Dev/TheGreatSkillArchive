# Recruitment & Aftercare

## Pre-Workshop Recruitment

### Landing Page Blurb

> Want to stop fearing the terminal and actually start shipping with Git? Join our hands-on GitHub in the Terminal workshop — zero slides, zero GUIs, just real commands and real repos.

### Who Should Enroll

- Developers who feel lost when they see `git merge`
- Designers / PMs who want to understand branches and PRs
- Open-source newcomers intimidated by the CLI

### Who Should Skip

- People who already authored a well-crafted `git rebase` explanation (look at advanced extension exercises)

---

## Aftercare — Keeping Momentum

1. **Buddy System**: Pair graduates for their firsts: first PR, first merge conflict, first Actions run.
2. **Weekly “Terminal Tip”**: 3-minute Slack/email blast with one new thing.
3. **Mini-Challenges**: One repo challenge per week escalating slightly each time.
4. **Office Hours**: 1h weekly drop-in to ask anything.
5. **Show-and-Tell**: Learners present their repositories monthly.
6. **GitHub Student Developer Pack**: Encourage enrollment for extra tooling.
7. **Contribution Map**: Track everyone’s open-source PRs in a shared doc.
