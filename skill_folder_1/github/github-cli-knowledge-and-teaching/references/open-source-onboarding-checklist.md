# Open-Source Onboarding Checklist

## Before Contributing
- [ ] Read README and CONTRIBUTING.md
- [ ] Check CODE_OF_CONDUCT.md
- [ ] Set up dev environment per instructions
- [ ] Run tests and confirm they pass

## First Contribution
- [ ] Find “good first issue”
- [ ] Comment to express interest
- [ ] Fork repo and create topic branch
- [ ] Make minimal, focused change
- [ ] Open PR with clear description
- [ ] Respond to review feedback promptly

## Long-Term
- [ ] Review one PR
- [ ] Help triage issues
- [ ] Update documentation
- [ ] Become a maintainer if you wish
