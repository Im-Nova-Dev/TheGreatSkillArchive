# Writing Good Commit Messages Workshop

## Structure
```
Type(scope): subject

Body (optional, wrapped at 72 chars)
```

## Types
- feat, fix, chore, docs, style, refactor, test, perf

## Exercise
Take 5 messy commit messages and rewrite them.

## Checklist
- One sentence subject
- Body explains why, not what (code shows what)
- Present tense
