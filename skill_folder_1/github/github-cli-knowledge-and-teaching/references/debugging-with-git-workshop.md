# Debugging with Git Workshop

## Tools
- `git bisect`
- `git blame`
- `git log -S` (pickaxe)

## Scenarios
1. “A bug appeared in the last 20 commits. Find the one that introduced it.”
2. “Who changed this line, and can we see the original?”
3. “When did `process()` stop being called?”

## Outcome
- Learners use Git as a debugging tool, not just a save tool
