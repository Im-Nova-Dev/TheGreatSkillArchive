# Pair Interview Guide — Git and GitHub Knowledge

Use this when evaluating practical Git knowledge in a live or recorded session.

## Setup

- Create a repo
- Ask candidate to perform tasks verbally or on screen
- Observe command recall, recovery habits, and mental model explanations

## Warm-Up Questions

1. Walk me through what `git status` shows right now.
2. How do you save code changes locally?
3. How do you send code to GitHub?

## Core Tasks

1. Create a branch and make two commits with real messages
2. Push and open a PR
3. Describe the diff and review checklist you would use
4. Merge and delete the branch
5. Show how to undo the last commit without discarding changes
6. Show how to recover a commit after a reset mistake

## Discussion Questions

1. When do you use `rebase` versus `merge`?
2. What does squash merge mean, and when would you use it?
3. How do you keep a fork in sync?
4. What signals would make you suspect a CI failure is environment-related instead of code-related?
5. How do you handle a merge conflict on a shared branch?

## Scoring Guidance

- Strong: explains why, not just how
- Adequate: remembers commands with minor prompting
- Needs support: confused about local vs remote or staging vs commit
