# GitHub CLI Teaching — Ops Layer

Use this skill for instructor-facing and operational material for Git/GitHub classes. Learner-facing exercise content lives in `github-cli-teaching-core`.

## What this skill is for

- Running workshops or cohorts
- Instructor onboarding, office hours, and cadence
- Tracking learner progress and feedback
- Live demo scripting and timing
- Certificates, badges, recognition, and alumni care
- Marketing, sponsors, and stakeholder reporting
- Accessibility, localization, and delivery logistics

## Related skills

- Load `github-cli-teaching-core` for learner-facing exercises and content.
- Load `github-pr-workflow`, `github-code-review`, `github-issues`, or `github-repo-management` for operational GitHub tasks.

## Teaching rule (one-skill-per-file)

Keep heavy skills split. Place session-specific detail in `references/*.md`. Do not overload a single SKILL.md with learner material AND admin material AND reference docs. Future skill reviews should preserve this shape by default.

## Split architecture (post-2026-06-03)

This skill now focuses on **instructor/ops materials**: cohort schedules, live demo scripts, admin templates, and workshop logistics.

For learner-facing teaching content, exercises, and student handouts, load `github-cli-teaching-core` instead.

Use `github-cli-knowledge-and-teaching` when: you are planning a course, managing cohorts, generating quiz answer keys, or running instructor-led sessions.
Use `github-cli-teaching-core` when: you are teaching a git/GitHub concept, running a practice drill, or answering "how do I explain X to a student?"
