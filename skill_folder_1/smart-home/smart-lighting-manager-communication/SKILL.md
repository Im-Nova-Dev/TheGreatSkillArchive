# Manager Communication for Lighting Techs

## Framing concerns
Use this structure:
1. State outcome risk.
2. State constraint.
3. Offer an alternative.
Example: "A simple switch-only install here risks cutting power to smart bulbs; we should install a Pico remote so users retain app control."

## Accountability language
Avoid blame; describe sequence.
Bad: "They installed it wrong."
Good: "The switch was left in the 'off' state, which prevents the bulb from appearing online. Household needs a control override plan."

## Pushing back on bad requests
- Smart bulbs on 3-way dumb switches: unsafe for bulb health.
- RGB in humid bathroom without damp-rated fixtures: code violation.
- lantic motion sensors pointing at HVAC: false positives guaranteed.
Document pushback with photo or note in install ticket.
