# Debugging Strategies

## Core topics

1. Reproduction
   - Minimal test case
   - Inputs, environment, timing

2. Isolation
   - Binary search bisect
   - Diff working vs broken

3. Inspection
   - Logging and trace points
   - Debuggers and breakpoints
   - Stack traces and memory dumps

4. Hypothesis testing
   - State expected invariant
   - Change one variable at a time

## Teaching approach
- Present one broken program per exercise.
- Force explanation before editing.
- Fix the cause, not the symptom.
