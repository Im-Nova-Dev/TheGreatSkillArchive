# Requirement Elicitation

## Core topics

1. Discovery
   - Interviewing stakeholders
   - Asking the right questions

2. Specification
   - User stories
   - Acceptance criteria

3. Validation
   - Ambiguity and edge cases
   - Confirmation and signoff

## Teaching approach
- Interview one stakeholder role.
- Turn one vague request into clear story.
- Identify one missing nonfunctional requirement.
