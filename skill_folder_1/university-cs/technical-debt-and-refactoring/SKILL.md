# Technical Debt and Refactoring

## Core topics

1. Debt
   - Identifying hotspots
   - Intentional versus accidental debt

2. Refactoring
   - Small safe steps
   - Tests before changes

3. Communication
   - Proposing cleanup work
   - Measuring impact

## Teaching approach
- Identify one debt hotspot in sample code.
- Refactor one function while preserving behavior.
- Write one short rationale for change.
