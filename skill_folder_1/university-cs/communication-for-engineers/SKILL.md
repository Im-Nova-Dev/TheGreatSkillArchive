# Communication for Engineers

## Core topics

1. Writing
   - Concise updates
   - Structured design explanations
   - Bug reports

2. Speaking
   - Explanations for nontechnical listeners
   - Presenting tradeoffs

3. Collaboration
   - Asking effective questions
   - Reviewing and receiving feedback

## Teaching approach
- Rewrite one vague update to clear update.
- Practice explaining one algorithm to nonengineer.
- Roleplay one design decision meeting.
